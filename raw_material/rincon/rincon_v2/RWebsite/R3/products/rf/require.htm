RightFax Fax Server Requirements

The computer designated as the RightFax server must meet or exceed the following specifications:

  • Windows NT 4.0 Server SP6, Windows 2000 Server (recommend SP2 or higher), or Windows XP Server (when released).
  • Pentium II 233 MHz or better processor.
  • 128 MB of RAM. More RAM may be needed if the server will also run other services, such as SQL Server, SMS or Microsoft Exchange.
  • 1 GB hard drive or better. More disk space may be needed depending on a number of factors, including how long users are allowed to keep old faxes in the system. The RightFax software itself requires 611 MB of disk space, but consider that each fax image (page) stored by RightFax consumes an average of 35 KB. If disk storage space is a problem, users should be encouraged to delete their faxes after reading or printing them.
  • VGA adapter and monitor. This is required for Windows NT.
  • Network interface card. The RightFax server must be connected to the local area network via a network interface card. RightFax will work on most popular LAN topologies and is compatible with most manufacturers' network interface cards. Any networking protocol supported by Microsoft networking may be used:  NetBEUI, TCP/IP or SPX.
  • Microsoft Internet Explorer 5 (or higher) must be installed on the server. If it is not already installed on the RightFax server machine, Internet Explorer will be installed as part of the RightFax server installation.

Note:

RightFax is an enterprise application that many companies depend on as much or as an alternative to e-mail. When e-mail is down fax becomes a very viable alternative for important/immediate communication, therefore putting fax and email servers on the same box is not recommended.


In order to use SSA conversion, you must disable the Exchange Events Service if RightFax and Exchange are on the same box. This is explained in the Administrator's Guide for RightFax versions 6.0 and later.

 

Fax boards take slots, IRQ's and I/O's that you might need for other boards or applications - it is entirely possible to run out of resources.


We do not recommend putting RightFax on a PDC or BDC in an NT 4.0 domain or an ADC in a Windows 2000 domain, as there are some random, anomalous events which can occur that are associated with the passing of security tokens and account validation when you run service accounts on a PDC/BDC. RightFax needs to validate users as service accounts.


We do not recommend other enterprise-wide applications such as SQL Server to be placed on the same box as RightFax as it is designed to monopolize all of the processor threads as necessary. RightFax can be a very processor intensive background application and it requires processor threads to process all of the requests to our database in a timely fashion.


Should MS Proxy Client be installed on the server, it will block the ports we use to communicate from EFM or from the client piece. The mspcint.ini specifies ports 1025-5000.


If using 8 or more fax channels in a single chassis and these fax channels are busy sending or receiving faxes more that 60% of the time, you should dedicate the server to RightFax.

RightFax Server with Integrated Module System Requirements

  • RightFax Enterprise Server v7.0 or higher.

  • Windows NT 4.0 Server SP6, Windows 2000 Server (recommend SP2 or higher), or Windows XP Server (when released)

  • Pentium 266 MHz processor or better.

  • 256 MB RAM or greater.

  • 4 GB of uncompressed usable hard drive space in a fault-tolerant disk configuration.

  • Software or hardware RAID level 1 minimum.

  • Appropriate number and type (PCI or ISA) of full-length expansion slots.

  • Network interface card. The RightFax server must be connected to the local area network via a network interface card. RightFax will work on most popular LAN topologies and is compatible with most manufacturers' network interface cards. Any networking protocol supported by Microsoft networking, including NetBEUI, TCP/IP or SPX, may be used.

  • CD-ROM.

  • SVGA display card with monitor settings of at least 800 x 600 dpi resolution and 256 colors.

  • 33.6 Kbps or faster modem (required for production technical support).

  • Microsoft Internet Explorer 5 (or higher) must be installed on the server.  If Internet Explorer is not already installed on the RightFax server machine, it will be installed as part of the RightFax server installation.

  • pcANYWHERE 32 v8.0 (required for production technical support).

  • Local administrator rights to all RightFax servers.

RightFax Fax Board Requirements

RightFax supports Brooktrout and Dialogic fax boards only. At least one fax board must be installed in the machine where the fax server software is running.

RightFax Phone Line Requirements

RightFax can use several types of phone lines: loop-start (regular analog phone lines), Direct Inward Dial (DID), Integrated Services Digital Network (ISDN), and several types of digital lines.
