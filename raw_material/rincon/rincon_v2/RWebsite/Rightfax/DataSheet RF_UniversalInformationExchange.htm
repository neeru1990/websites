HOW IT WORKS

Within your organization a variety of applications are used in order to generate information critical to running a business. These include applications that employees use for daily communication such as email and desktop, or back-office applications including ERP, CRM, Host and legacy as well as other network applications implemented to bring efficiencies such as Document Management, Archival, Imaging and Forms applications. RightFax Information Exchange leverages these applications by electronically exchanging the information they generate with trading partners.

Figure 1: Each application outputs various forms of “information.” The RightFax flexible API and application integration captures and receives this information seamlessly.

Once information is sent from application(s) to RightFax, a four-phase workflow process begins.

RightFax Universal Information Exchange Automating Business Processes

Figure 2: RightFax incorporates a four phase workflow process that captures any form of information, renders it into an electronic image, distributes the information via fax, email or over the Internet, and creates customized reporting, including host notifications.

Phase I: Data Recognition
Whether sending a large batch of data from a host application, a Java file from an ERP or CRM application, or a simple word document or email, RightFax integration solutions are able to capture and recognize data.

Without having to change your existing applications, RightFax data recognition technology will capture and configure data being sent from applications in the following formats:

  • File (PCL, Text, PS)
  • LPR Connection, Print
  • Serial
  • 3270
  • TCP/IP
  • File Transfer Protocol (FTP)
  • JAVA
  • XML
  • MQ Series
  • SAP Connect
  • SAP Script
  • Oracle Print Files
  • Embedded Codes
  • Inbound Fax

This extensive list allows Captaris RightFax to integrate with virtually any application.

Phase II: Information Assemby
After receiving the data, RightFax assembles the information by rendering an electronic image of each separate document. Depending on the file type and data format, this can be accomplished in a number of ways - one-off documents or emails can be rendered into TIFF images, "Filters" can be created using the new RightFax Wizard. In instances when codes have been embedded into the data stream, the forms overlay process is implemented.

Figure 3: FCL Commands can be embedded into data streams or wrapped around data streams to create electronic images of business documents.

Phase III: Information Distribution
Once the electronic images have been created, RightFax delivers your information securely and instantaneously via fax, email or over the Internet. Whether sending a hundred or tens of thousands of documents a day, our solutions are built to scale and handle any capacity. And with advanced features such as load balancing, least cost routing and scheduled delivery, additional cost efficiencies are achieved.

Phase IV: Reporting
RightFax stands above other e-document delivery solutions with extensive reporting capabilities, including:

  • Standard reports that monitor all e-document delivery activity
  • The ability to create customized reports including graphs
  • Notification of information delivery
  • Highly customizable host notifications including document level notifications, user level notifications and host notifications to a database

FEATURES

The RightFax Integration Wizard helps to quickly implement RightFax Integrations including:

  • Defining the protocol to communicate with the RightFax Server and information that appears on each sent document
  • Defining the settings for receiving data from host, legacy and ERP applications
  • Defining actions for alerting senders of the status of sent documents
  • Designing forms that are combined with data to create documents
  • Over 100 powerful FCL (Facsimile Command Language) commands that allow customization of RightFax Information Exchange integration to best suit the host environment
  • Certified integrations with Oracle, SAP and Baan applications
  • Supports the following host systems: IBM, DEC, HP, UNISYS, Sequent, Wang, NCR and others
  • Supports the following connectivity options: LPR, 3270, TCP/IP, FTP, 5250, IBM MQSeries, Client v5.1 for Windows NT, Serial IPX/SPX, Directory scan

MINIMUM SERVER REQUIREMENTS

  • Windows NT Server v4.0 service pack 6 or later, or Windows 2000 servers
  • Pentium II 233 MHz processor
  • 128 MB of RAM. More RAM may be needed if the server will also run other services such as SQL Server, SMS or Microsoft Exchange
  • 1 GB hard drive or larger
  • Network interface card, any networking protocol supported by Microsoft networking: NetBEUI, TCP/IP or SPX
  • Microsoft Internet Explorer 5. If it is not already installed, Internet Explorer will be installed as part of your RightFax server installation
  • VGA Display adapter and monitor (required for Windows NT)
  • CD-ROM Drive
